使用Hexo加Github Pages搭建简洁的个人博客

无标签
发布日期:   2019-08-17
文章字数:   1.5k
阅读时长:   5 分
阅读次数:  

系统环境

  • Ubuntu 19.04
  • Node.js v10.16.3
  • npm 6.9.0
  • Git 2.20.1

本地安装

安装相关工具－－Nodejs, npm及git

安装Nodejs及npm

使用Ctrl+Alt+T快键键打开终端,先同步软件仓库

1
sudo apt update

接着，使用以下命令安装nodejs

1
sudo apt install nodejs

最后，检查是否安装成功

1
nodejs -v

npm是nodejs的包管理工具，安装的方法与上面类似

1
sudo apt install npm

同样，可以通过下面的命令检查是否安装成功

1
npm -v

当然还可以选用PPA的方式安装最新版本的nodejs及npm.如果可能要使用多个版本的nodejs,则可安装nodejs的版本管理工具NVM．建议在通过这些方式安装前先使用sudo apt remove命令卸载掉已安装的nodejs及npm.

不过如果只是使用hexo,建议按上面的方式安装就可以了

git

git是目前最常用的版本管理工具，主要由linus大神开发，在许多地方都会用到．

1
sudo apt install git

类似的用一下命令检查是否安装成功．

1
git --version

安装Hexo

在ubuntu19.04版本上，如果按照官网的指南,输入命名npm install hexo-cli -g,就会出现下面的报错．

这个主要是权限问题，加上sudo即可.即输入下面的命令

1
sudo npm install hexo-cli -g
  • 注：官方其实并不推荐这种做法，更好但也更复杂的方法可以参考官方的指南

安装时还是会有WARN,但并不影响hexo的安装，可输入下面的命令检查是否安装成功

1
hexo -v

测试

安装好hexo后，我们可以先在本地测试一下，先创建一个文件夹用于测试：

1
2
mkdir ~/test
cd ~/test

创建一个博客系统，并初始化

1
2
3
hexo init blog
cd blog
npm install

在本地测试运行效果

1
2
3
hexo clean
hexo g
hexo s

命令行的显示如下：

按住Ctrl键，点击http://localhost:4000,便会在浏览器中打开如下网页，便说明安装成功．

部署到Github Pages

到github官网,点击Sign Up,根据提示一步步填写即可．需要邮箱验证．

github账号的申请

首先新建一个项目

注意格式为用户名.github.io,项目创建完成后，点击进入Setting选项

Source 一栏选择master分支，再选择一个主题，就可以githubPages的设置．现在你就可以通过https://youname.github.io访问你的github Pages了．（如果想换一个域名，绑定自己的域名，在自己注册的域名服务商添加一条CNAME解析即可）

设置ssh登录秘钥

因为我们要将hexo 生产的静态博客内容直接通过ssh推送到github仓库，所以需要首先让github网站接受我们的登录请求．

本地生成公钥秘钥

1
2
3
git config --global user.name "用户名"
git config --global user.email "你的github账户邮箱"
ssh-keygen -t rsa -C "你的github账户邮箱"

最后一条命令会在~/.ssh文件夹中生成公钥秘钥文件，接着我们可以使用下面命令查看公钥内容

1
cat ~/.ssh/id_rsa.pub

然后需要我们将公钥粘贴到github中．在Setting选项中打开SSH and GPG keys选项

新建一个Key, 名字随便，内容即为刚刚粘贴的公钥．

最后在本地终端中输入如下命令：

1
ssh git@github.com

如果显示与下面类似，就说明设置成功了．

将本地的Ｈexo博客内容推送到github仓库

当准备正式建立博客站时，需要建立一个正式的文件夹来放正式的blog相关的内容，步骤和测试时一样．

1
2
3
4
5
mkdir ~/myBlog
cd ~/myBlog
hexo init blog
cd blog
npm install

同样采用下面的三个命令测试是否安装成功

1
2
3
hexo clean
hexo g
hexo s

接着找到blog目录下的_config.yml文件．修改deploy相关内容．

repository改一下对应的用户名．type对应git, branch 对应master,这里有两个坑需要注意一下．一个是常见的分号:需要为英文状态下的分号:．第二个是:后面要跟一个空格，这个是yaml文件格式的要求．

修改完成后保存．接着在blog目录下使用hexo d命令，一般会要求输入github的用户名和密码．如果没有任何反应或是报错，可以先安装hexo-deployer-git后

1
npm install hexo-deployer-git --save

再次推送到github试试．

Hexo的个性化设置

经过上面的步骤基本就完成了一个博客的搭建．在本地可以通过hexo new "article_name"新建文章，然后再在blog目录下的source/_posts目录中编辑对应文章的markdown文档，然后再hexo clean &&hexo g生成对应的静态文件．hexo s本地预览．hexo d推送到网站上．这样所有的人就可以通过youname.github.io看到你的文章了（一般其实也就你自己看：）

简洁，方便．使人将更多的精力放在写作本身上．当然一段时间后，你想稍稍个性化一下自己的博客，使之看起来像是一个实实在在的个人博客，而不是hexo的一个官方demo.下面是一些常用的个性化设置.

安装主题

Hexo官方的主题是landscape, 如果想换一下其他主题可以到hexo的官方主题板块去看一看，这里以比较流行的Next主题为例(官方github链接),

首先cd到blog目录下．然后用下面的命令将主题文件clone到本地

1
git clone https://github.com/theme-next/hexo-theme-next themes/next

接着修改_config.yaml中的theme修改为next即可．
