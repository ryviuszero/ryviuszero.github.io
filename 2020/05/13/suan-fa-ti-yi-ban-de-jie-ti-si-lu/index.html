什么是算法

In mathematics and computer science, an algorithm is a finite sequence of well-defined instructions, typically used to solve a class of specific problems or to perform a computation.

---------- wiki: Algorithm

广义上来讲解决任何问题的特定思路，方法都可以被称为算法，当然算法一般是和计算机科学息息相关，所以一般会将问题定义成：给定一个输入，得到一个符合要求的输出。而算法则是实现输入到输出映射的步骤。为了计算机能够真正解决问题，还需要将算法转化成程序，这就需要数据结构的配合，来实现算法中对数据的各种操作。虽然问题各异, 但是解决思路却是相似的。

暴力算法

暴力的意思是先不去考虑任何优化的技巧或思路，以解决问题为第一要务，用直观的思路来实现算法。穷举和模拟是两种常见的暴力算法思路。这种思路一般适合在自己没有其它更加高效的方法的时候使用，作为之后优化的基础，或者是得到几组基础解后来作为找规律的基础（对于有特定规矩的数学问题特别有用）。

穷举法

Leetcode 1: 两数之和

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。

你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

示例:

给定 nums = [2, 7, 11, 15], target = 9

因为 nums[0] + nums[1] = 2 + 7 = 9
所以返回 [0, 1]

这是 leetcode 的第一题，在许多 OJ 中也是用来熟悉评测环境的一题，第一反应用双重循环遍历枚举所有可能即可，代码如下：

class Solution {
    public int[] twoSum(int[] nums, int target) {
        int N = nums.length;
        int[] ans = new int[2];
        
        for(int i = 0; i < N ; i++)   {
            for(int j = i +1; j < N ; j++){
                if(nums[i] + nums[j] == target){
                    ans[0] = i;
                    ans[1] = j;
                    return ans;
                }
            }
        }
        
        return ans;
    }
}

上面暴力枚举的方式的时间复杂度为 O(n^2^)，当然还有改进的方法，就是用哈希表先将 sum - numOne 存起来，一边存一边查表，即可将时间复杂度优化到 O(n). 代码如下：

class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer,Integer> map = new HashMap<>();
        for(int i = 0; i < nums.length; i++){
            int complement = target - nums[i];
            if(map.containsKey(complement)){
                return new int[]{map.get(complement), i};
            }
            map.put(nums[i],i);
        }
        throw new  IllegalArgumentException("No two sum solution");
    }
}

模拟

有的问题并没有明显的特点及结构，或者自己一时想不出什么思路，就可以先不要去追求优化的方法，先就按题目的意思，模拟题目的操作，直接将题目叙述转化为代码。先确保得到一个可行的答案，再看看有没有优化的地方。

Leetocde 874: 模拟行走机器人

机器人在一个无限大小的网格上行走，从点 (0, 0) 处开始出发，面向北方。该机器人可以接收以下三种类型的命令：

-2：向左转 90 度
-1：向右转 90 度
1 <= x <= 9：向前移动 x 个单位长度
在网格上有一些格子被视为障碍物。

第 i 个障碍物位于网格点  (obstacles[i][0], obstacles[i][1])

如果机器人试图走到障碍物上方，那么它将停留在障碍物的前一个网格方块上，但仍然可以继续该路线的其余部分。

返回从原点到机器人的最大欧式距离的平方。

解答

class Solution {
    public int robotSim(int[] commands, int[][] obstacles) {
        int x = 0, y = 0, max = 0, index = 0;
        Set<String> set = new HashSet<String>();
        
        //序列化障碍物的位置
        for(int[] arr : obstacles){
            set.add(arr[0] + "-" +arr[1]);
        }

        //四个方向， 上右下左
        int[][] directions = {{0,1}, {1,0},{0,-1},{-1,0}};

        for(int n: commands){
            if( n == -1){
                index++;
                if(index == 4){
                    index = 0;
                }
            } else if(n == -2){
                index--;
                if(index == -1){
                    index = 3;
                }
            }
            else {
                while(n-- > 0 && !set.contains((x+directions[index][0]) + "-" + (y +directions[index][1]))){
                    x += directions[index][0];
                    y += directions[index][1];
                }
            }
            max = Math.max(max, x*x + y*y);
        }
        return max;
    }
}

减而治之与分而治之

减而治之就是每次将问题的规模缩小一个常数，最终以时间复杂度为 O(n) 的代价解决问题，而分而治之则是每一步都将问题分为几个规模相同的子问题。两者的年点都是找到切分的方法及如何实现问题规模的缩小，需要注意的点就是最后最小的平凡子问题。减而治之思想的运用如颠倒整个数组等，而快排，二分查找都是分而治之思想的运用。

Leetcode 189: 旋转数组

给定一个数组，将数组中的元素向右移动 k 个位置，其中 k 是非负数。

示例 1:

输入: [1,2,3,4,5,6,7] 和 k = 3
输出: [5,6,7,1,2,3,4]
解释:
向右旋转 1 步: [7,1,2,3,4,5,6]
向右旋转 2 步: [6,7,1,2,3,4,5]
向右旋转 3 步: [5,6,7,1,2,3,4]

由于题目要求使用原地算法，一种思路就是多用几个变量保证前一个数移到后一个数的位置上时，前一个数已经取出，这个每几个数会形成一个环，当一个环移完后就移动另一个环，直到全部就位。

class Solution {
    public void rotate(int[] nums, int k) {
        int N = nums.length;
        
        k = k % N;
        
        int count = 0;
        
        for(int start = 0; count < N; start++){
            int curr  = start;
            int pre = nums[curr];
            do{
                int next = (curr + k) % N;
                int tmp = nums[next];
                nums[next] = pre;
                pre = tmp;
                curr = next;
                count++;
            }while(curr != start);
        }
        
        
    }
}

不过这题还有一种巧妙的思路，那就是先将所有数颠倒，再颠倒前 k 个数，再颠倒后 n - k 个数，也可以得到正确结果。

class Solution {
    public void rotate(int[] nums, int k) {
        k %= nums.length;
        
        reverse(nums, 0, nums.length -1);
        reverse(nums, 0, k -1);
        reverse(nums, k , nums.length -1);

            
    }
    
    public void reverse(int[] arr, int start, int end){
        while(start < end){
            int tmp = arr[start];
            arr[start] = arr[end];
            arr[end] = tmp;
            start++;
            end--;
        }
    }
    
}

至于分而治之，二分查找就比较经典。

比如 leetcode 35 搜索插入位置

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

你可以假设数组中无重复元素。

示例 1:

输入: [1,3,5,6], 5
输出: 2

答案：

class Solution {
    public int searchInsert(int[] nums, int target) {
            int left = 0, right = nums.length -1;
            while(left <= right){
                int mid = (left + right)/ 2;
                if(nums[mid] == target){
                    return mid;
                }
                else if(nums[mid] < target ){
                    left = mid + 1;
                }
                else {
                    right = mid - 1;
                }
            }

            return left;
    }
}

动态规划

适用于具有最优子结构的问题，即当前的状态可以由过去的几个状态推导出来。难点在于找到状态转移方程。动态规划中经典的问题就是背包问题。这个可以找找著名的“背包九讲”

例子：198. 打家劫舍

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你在不触动警报装置的情况下，能够偷窃到的最高金额。

示例 1:

输入: [1,2,3,1]
输出: 4
解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。
     偷窃到的最高金额 = 1 + 3 = 4 。

答案

class Solution {
    public int rob(int[] nums) {
        if(nums == null || nums.length == 0) return 0;
        int n = nums.length ;

        int[] dp = new int[n + 1];
        dp[0] = 0;
        dp[1] = nums[0];

        for(int i = 2; i <= n; i++){
            dp[i] = Math.max(dp[i - 2] + nums[i - 1], dp[i - 1]);
        }

        return dp[n];
        
    }
}

贪心思想

每一步的选择都是当前最优，最后仍能得到全局最优的算法。难点在于找到问题对应的贪婪策略。

例子：1221. 分割平衡字符串

在一个「平衡字符串」中，'L' 和 'R' 字符的数量是相同的。

给出一个平衡字符串 s，请你将它分割成尽可能多的平衡字符串。

返回可以通过分割得到的平衡字符串的最大数量。

 

示例 1：

输入：s = "RLRRLLRLRL"
输出：4
解释：s 可以分割为 "RL", "RRLL", "RL", "RL", 每个子字符串中都包含相同数量的 'L' 和 'R'。

答案：

class Solution {
    public int balancedStringSplit(String s) {
        int count = 0;
        int countL = 0, countR = 0;
        int n = s.length();
        int i = 0;
        
        while(i < n){
            if(s.charAt(i)=='R')
                countR++;
            else
                countL++;
            
            if(countR == countL){
                count++;
                countL = 0;
                countR = 0;
            }
            i++;
        }
        
        return count;
    }
}
